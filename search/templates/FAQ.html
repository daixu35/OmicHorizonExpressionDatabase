1. What does the Omic Horizon Expression database do?

Omic Horizon Expression (or OmicHorizon@Expression) is a database of gene expression in multiple tissues, primary cells and cell lines based on omics data. It aims to show mRNA expression levels by TPM values from bulk RNA sequencing (RNA-Seq) data for multiple species. However, only rat samples are included in the current version. These samples are divided into three categories, including tissues, primary cells and cell lines. Some tissues are further divided into several different subtissues according to the anatomical site. As an omic resource, the database provides horizons of gene expression across various conditions.

2. Where are the data come from?

We collected rat RNA-Seq data from the Sequence Read Archive (SRA) database. The downloaded sra format files were uniformly processed, including file format conversion, quality control, reads mapping (reference genome: mRatBN7.2) and gene expression quantification by TPM values. The current version contains over 2,600 samples, which are categorized into 48 tissues, 40 primary cell types, and 25 cell lines.

3. How can the database be queried?

The OmicHorizon@Expression database supports requests using gene symbols (or alias), Ensemble and Entrez gene IDs. The interface is simple and easy to understand. A basic operation and work flow are shown as follows:
(1) Select a search type, such as tissue, primary cell and cell line, from the drop-down menu. Then, an interesting tissue or cell type is selected before the new drop-down menu pops up for further selection of subtissue or cell sources. When ‘All’ is selected, all tissues or cells under the layer will be queried to show the result.
(2) Enter your query gene symbol or gene ID, and click on the button ‘GO’ to enter the result page for the gene expression profile.
(3) Please note that not all legends on the X-axis will be displayed when there are too many retrieved tissues or sub-tissues, for example, when "All" is selected in the first step.

4. How to understand the query result?

The result page includes three sections:
Section 1. Basic information of the queried gene, such as gene description, gene alias, gene IDs, gene biotype and chromosome location.
Section 2. Gene expression profile shown in a box plot. In the plot, the x-axis represents the queried tissues, primary cells or cell lines, while the y-axis represents the expressional TPM values (see below). A box plot consists of six data nodes, which arrange a set of data from largest to smallest, and calculate its upper edge (Q3+1.5*IQR), Q3 (75th percentile), median (50th percentile), Q1 (25th percentile), lower edge (Q1-1.5*IQR) and abnormal values in circle in the plot. The upper and lower edges are set to the maximum and minimum values, respectively, when there are no abnormal values. Several prompt messages by mouse are shown in the following figure.
Section 3. A table form shows the gene expression values of the queried gene in the interested tissues or cells. Each value represents a sample source.
The box plot and table files are downloadable and can be saved to a hard drive.

5. What is a TPM value?

To eliminate the effects of sequencing depth and gene length, we used the TPM (transcripts per kilobase million) value to measure the gene expression level. A TPM value is calculated by the following formula:
TPM=(Ni/Li)*1,000,000/sum(Ni/Li+…….+ Nm/Lm)
Ni: the count of reads mapping to gene i
Li: the length of exons of gene i
